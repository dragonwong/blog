两天在做代码的排版遇到了他俩，好奇研究了下。

<pre>

官方解释：

The <pre> element represents a block of preformatted text.

<pre>代表一块预编排的文本。这里可以得知两个信息，一是<pre>是块级元素，二是<pre>会保留原有格式。

看看chrome的预置样式：

pre, xmp, plaintext, listing{
    display: block;
    font-family: monospace;
    white-space: pre;
    margin: 1em 0px;
}

<pre>保留原有格式的秘密就在于white-space: pre;这条样式里，传送门：CSS/Properties/white-space。

<code>

官方解释：

The <code> element represents a fragment of computer code.

我觉得<code>比<pre>更加语义化，他直接对外宣称“这是一段计算机代码”。

看看chrome的预置样式：

tt, code, kbd, samp {
    font-family: monospace;
}

很简单，只是让他以等宽字体显示。

我们经常会让<pre>和<code>一起使用，由于<pre>会保留换行，所以<code>后的换行也会被保留，导致页面展示时代码开头会多出一个空白行，对此，不得不在编辑时删除<code>与代码之间的空格。下面是对<pre>和<code>一些情况的截图，其中绿色背景是<pre>，黄色背景是<code>。
