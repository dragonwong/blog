String to Number

var num;
//method_1
num = +'1';
//method_2
num = Number('1');
//method_3
num = parseInt('1');	//or parseFloat()

The efficiency of method_2 and method_3 are similar while method_1 is a bit more efficient than them.

Efficiency: +'1' > Number('1') = parseInt('1')

Number to String

var str;
//toString
str = (1).toString();
str = 1..toString();
str = 1.0.toString();
str = Number.prototype.toString.call(1);
str = 1['toString']();
//plus
str = '' + 1;
//String
str = String(1);

Efficiency: (1).toString() > '' + 1 = 1..toString() = 1.0.toString() > 1['toString'](); > Number.prototype.toString.call(1) = String(1)

Hey! What's 1..toString? Do you mean 1.toString?

Please read the following.

Convert a literal number to String

var num = 1;
num.toString();	//we will get 1
1.toString();	//we will get SyntaxError: Unexpected token ILLEGAL

The thought is right but it is wrong in syntax. In 1.toString();, 1 is a literal number. When the interpreter reads the dot after 1, it expect another number to compose a decimal number. So the dot in 1.toString(); is not a dot of attribution, but a decimal point!

Now, you known why we need another dot, so we get 1..toString. Er, it looks a little strange indeed. We can use 1.0.toString();or other methods mentioned above.
